# CLAUDE.md — claude-code-lecture

> 매 세션 자동 로드. 목표 길이 100~200줄, 넘으면 잘라낼 것부터.
> 코드/파일 구조를 보면 알 수 있는 건 적지 않는다.

마지막 업데이트: 2026-06-28

---

## 1. 이 레포의 정체 (가장 자주 헷갈리는 것)

- 이건 **"실리콘밸리 엔지니어의 Claude Code" 강의 실습 워크북**이다.
- 안에는 학습자가 직접 채울 **빈 양식(templates)** 만 있다. 동작하는 앱이 아니다.
- 빌드·테스트·실행 명령이 **없다** (순수 마크다운). `npm`·`build` 찾지 마라.

## 2. 절대 규칙

- 템플릿 파일(`exercises/templates/*.md`)의 **양식 구조(섹션 제목·표 틀)를 임의로 바꾸지 마라.** 학습자가 채울 빈칸이다. 요청받았을 때만 내용 작성.
- 강의 본문/슬라이드는 이 레포에 없다 — 외부 강의 사이트에 있다. 없는 파일을 지어내지 마라.
- 한글 경로·파일명을 ASCII로 바꾸지 마라.

## 3. 디렉토리 규칙

```
Part<N>-<한글주제>/Ch<NN>-<한글주제>/exercises/templates/*.md
```

- 새 실습 양식은 항상 해당 챕터의 `exercises/templates/` 아래.
- 루트 `README.md` = 레포 안내. 챕터를 추가하면 여기 목차도 갱신.

## 4. 작업 시 기본기

- 커밋: Conventional Commits (`docs(ch02): ...`, `feat: ...`). 최근 히스토리 스타일을 따른다.
- 변경 전 `git status`로 현재 위치 확인. 기본 브랜치는 `main`.
- 마크다운만 다루므로 lint/빌드 대신 **내용 정확성**으로 검증한다.
